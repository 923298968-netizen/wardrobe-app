## 📱 App 上架技术方案

### 技术栈选择

#### 方案 1：Cordova/PhoneGap（推荐用于快速上架）

**优点：**
- 纯 Web 技术迁移，无需重写
- 支持跨平台（iOS + Android）
- 上架流程熟悉，文档完善

**步骤：**
1. 安装 Cordova CLI
2. 创建项目并导入现有代码
3. 添加平台：`cordova platform add ios android`
4. 添加必要插件：
   - cordova-plugin-camera（拍照上传）
   - cordova-plugin-geolocation（天气定位）
   - cordova-plugin-device（设备适配）
5. 打包：`cordova build ios` / `cordova build android`
6. iOS：用 Xcode 打开 platforms/ios，签名后上架
7. Android：生成 .apk/.aab，上传到 Google Play

#### 方案 2：Capacitor（Ionic 现代方案）

**优点：**
- Ionic 团队维护，更现代
- 更好的性能和开发体验
- 支持渐进式 PWA

**步骤：**
1. `npm init @capacitor/app`
2. 将现有代码放入 www 文件夹
3. 添加平台：`npx cap add ios android`
4. 同步：`npx cap sync`
5. 原生打包同 Cordova

#### 方案 3：React Native（长期推荐）

**优点：**
- 真正的原生体验
- 生态完善，组件丰富
- 性能最佳

**缺点：**
- 需要重写现有代码
- 学习成本高

**建议：** 先用 Cordova 快速上架验证市场反馈，后续考虑重写为 RN

## 🛠️ 技术实现建议

### 1. 本地存储升级

当前使用 localStorage，上架后建议：
- **iOS/Android**：使用 SQLite 插件
- **用户数据备份**：支持导出/导入 JSON 文件
- **云端同步**：可选付费功能（Firebase、Supabase）

### 2. 拍照功能增强

- **相机权限处理**：优雅降级
- **图片压缩**：避免存储过大
- **OCR 识别**：可考虑识别衣物标签（需要 AI 服务）

### 3. 离线优先

- Service Worker 已配置
- 预加载关键资源
- 离线缓存天气数据

### 4. 性能优化

- 图片懒加载
- 长列表虚拟滚动
- 动画使用 CSS transform
